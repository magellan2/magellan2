This is the editor environment of Extended Commands.

This text attempts to clarify the complex possibilities of this plugin. This editor is based on BeanShell and therefore similar to Java. There are classes and methods. The variables are more tipical.

You have the choice to edit one of the following types.

  • units - every single unit in the game
  • container - like regions, ships and buildings
  • library - a collection of useful functions for units and containers

The API for the individual classes is extensive. In principle one has control over all facets of Magellan and the most recently loaded report. In theory it is even possible to open custom windows, but this documentation will not go into that. For a deeper understanding of the API we recommend the following sites:

  • Magellan Library
    magellan.library.*
    Contains a lot of descriptions for the most common classes in Magellan.
  • Magellan Rules
    magellan.library.rules.*
    Contains some defintions for the rules.
  • Extended Commands Helper
    Helper API
    Contains all the method descriptions useful inside the helper tool.
  • Magellan2 API (complete)
    http://magellan.narabi.de/nightly-build/api
    Documentation on all classes and possibilities of the Magellan client. The classes in magellan.library should prove especially interesting.
  • Java 5 API
    http://java.sun.com/j2se/1.5.0/docs/api
    Documentation on the current Java environment. All these classes are at your disposal.
  • BeanShell Syntax
    http://www.beanshell.org/manual/contents.html
    Complete documentation on the possibilities of BeanShell.
