This is the editor environment of Extended Commands.

This text attempts to clarify the complex possibilities of this plugin. This editor is based on BeanShell and therefore similar to Java. There are classes and methods. The variables are more tipical.

At the moment you've selected a unit. The following global variables are available:

  • world - the most recently loaded report and all information connected with it (islands, regions, factions, etc.)
  • helper an auxiliary class with handy functions
  • unit the object representing the unit of type magellan.library.Unit

To be able to use additional classes from Java or Magellan, these have to be imported. This happens for example as follows: 

import magellan.library.*;

Region region = unit.getRegion();

The API for the individual classes is extensive. In principle one has control over all facets of Magellan and the most recently loaded report. In theory it is even possible to open custom windows, but this documentation will not go into that. For a deeper understanding of the API we recommend the following sites:
