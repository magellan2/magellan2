Der SplitPane-Modus

Was sind SplitPanes?

Zuerst sollte wohl erklärt werden, was SplitPanes sind:
SplitPanes sind graphische Komponenten, die eine Fläche in zwei Bereiche unterteilen. In diesen Bereichen könen wieder Komponenten dargestellt werden. Mittels Benutzerinteraktion oder durch Programmierung kann die Trennlinie dabei beeinflußt werden.

Vorteile des SplitPane-Modus

In diesem Modus ist es relativ einfach möglich, eine klar gegliederte und vom Benutzer anpassbare Oberfläche zu entwerfen. Die Größe der einzelnen Fensterelemente kann dabei während der Benutzung verändert werden.

Nachteile des SplitPane-Modus

Das Konzept der SplitPanes bietet bei allem leichten Aufbau auch einen gravierenden Nachteil: Die rekursiv aufgebaute Struktur ist relativ starr und nur in engen Grenzen vom Anwender anzupassen. Ohne den Grundaufbau zu ändern, ist es nicht möglich größere Änderungen der Darstellung vorzunehmen.

Konfiguration

Das wichtigste ist dabei natürlich die Konfiguration des Magellan Desktop. Die Entwicklern wählten die Speicherung in einer externen Textdatei, die den Namen "magellan_desktop.ini" hat und sich im Magellan-Verzeichnis befindet. Sollte keine solche Datei vorhanden sein, so startet der Desktop mit einer Standardkonfiguration und speichert diese zum Schluß in eine neu erzeugte Datei.

Der Aufbau der Konfigurationsdatei wurde dabei an die Standard-Ini-Struktur der verbreiteten Betriebssysteme und Software angelehnt: 

[SplitSet-Name]
SPLIT 0.5 H
SPLIT 0.15 V
COMPONENT MAP
COMPONENT DETAILS
/SPLIT
COMPONENT OVERVIEW
/SPLIT
[SplitSet-Name]
.
.

Wie zu sehen, erhält jedes "Set" von Konfigurationsanweisungen einen Namen, der in eckigen Klammern angegeben wird. Wichtig ist dabei, den Namen nicht mit "Layout_" zu beginnen, da dies ein Layout für den Layout-Modus kennzeichnet. Darauf folgen dann zeilenweise die Anweisungen an den Magellan Desktop.

Der Aufbau der Anweisungen ist der Einfachheit halber sowohl der Umsetzung im Programm als auch der ähnlichen FrameSet-Struktur der HTM-Sprache nachempfunden. Sollten Sie sich damit nicht auskennen, so benutzen Sie den folgenden Ablauf, um eine eigene Konfiguration zu "erfinden". Starten Sie dabei im ersten Schritt mit der Gesamtfläche, daß heißt dem gesamten Bildschirm:

  • Nehmen Sie sich Ihre Fläche, führen Sie eine der beiden folgenden Anweisungen aus
    • Erzeugung eines Splits = Einer Teilung
      • Teilen Sie diese in zwei Hälften, entweder horizontal (zwei Hälften nebeneinander) oder vertikal (zwei Hälften übereinander). Notieren Sie sich das Kennwort "SPLIT",das Teil-Verhältnis (zwischen 0 und 1) und die Orientierung ("H" für horizontal, "V" für vertikal)
      • Nehmen sie die linke/obere Hälfte und beginnen Sie beim ersten Schritt erneut (setzen sie für "Ihre Fläche" die linke/obere Hälfte ein).
      • Nehmen sie die rechte/untere Hälfte und beginnen Sie beim ersten Schritt erneut (setzen sie für "Ihre Fläche" die rechte/untere Hälfte ein).
      • Notieren Sie sich das Kennwort "/SPLIT"
    • Einsetzen einer Komponente
      • Notieren Sie sich das Kennwort "COMPONENT" und den Kennamen der Komponente, die in dieser Fläche angezeigt werden soll. Kennnamen siehe Tabelle unten.
    • Kehren Sie zu Ihrem letzten Schritt zurück (sollte kein letzter Schritt mehr bestehen, sind Sie fertig).

    Nun sollten Sie eine Notiz in den Händen halten, die Ihre eigene Kreation wiederspiegelt. Geben Sie diese einfach mit einem eigenen Namen ein. Sie könen Sich dabei am vorgegebenen Standard-Set orientieren.

    Wichtig sind natürlich die Kennamen für die möglichen Komponenten, die Magellan darstellen kann.


    Schlüsselwort Komponente
    ORDERS Befehlseditor
    DESCRIPTION Beschreibung einer Region/Gebäude/Schiff/Einheit, losgelöst vom Namensfeld
    DETAILS Anzeige der Details einer Region/Gebäude/Schiff/Einheit
    HISTORY Verlaufsanzeige, Liste der letztgewählten Objekte
    MAP Große Karte
    MESSAGES Nachrichtenanzeige
    MINIMAP Kleine Karte
    NAME Name einer Region/Gebäude/Schiff/Einheit, losgelöst vom Beschreibungsfeld
    NAME&DESCRIPTION Zusammenfassung von Names- und Beschreibungsfeld, horizontal getrennt
    OVERVIEW Regions-/Inselbaum
    OVERVIEW&HISTORY Zusammenfassung von Regionsbaum und Verlaufsanzeige, horizontal getrennt

    Die auswertbaren, daß heißt vom Magellan verstandenen Konfigurationen können dann im Menü "Desktop" im Unterpunkt "Split-Sets" unter dem von Ihnen angegebenen Namen ausgewählt werden.
