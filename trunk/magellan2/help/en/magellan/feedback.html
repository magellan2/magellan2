Feedback and Support

Bugs in Magellan

For us as Developers the error messages are a good start when tackling a problem. These can be found in the errors.txt file in the Magellan folder; please include these error messages if you have a problem with Magellan. To reproduce an error it may prove necessary to send in (an excerpt of) the used CR. These CRs are of course handled confidentially, and will be deleted after the issue has been resolved.

Mantis

Mantis (http://magellan-client.sf.net/mantis/) is a Webplatform for bugreports and featurewishes, that for these aspects is preferred over the mailing list, because it significantly simplifies the handling of reports. For use it is necessary to register, to be able to keep the reporter up-to-date on the bugreport or feature wish.


Featurewishes = further development

We try to develop Magellan according to the wishes of its users. Mantis helps enermously in this, because all feature wishes are centrally and publicly available. Additionally Mantis offers 'Votings', which means that every user can vote whether or not a certain new feature should be implemented. This makes it easy to recognize which wishes are important and should get a high priority.

Mailing list

There is a mailing list for Magellan, eressea-javaclient@eressea.kn-bremen.de, where all changes to Magellan are announced. After such CVS-mails Magellan is updated within minutes, so that you can download the new version. Through the mailing list anyone that is interested can join in the discussion about the program, ask questions about its use or request help with problems.

If you send a mail to the list without being subscribed to it you should mention this, because otherwise the answer will most likely just be sent to the list again, and not to the sender.

To subscribe to the list it is enough to send a mail to majordomo@eressea.kn-bremen.de with an arbitrary subject and

subscribe eressea-javaclient
end
in the mail body.

Newsgroup

Through newsgroup Eressea News-Server (news://eressea.upb.de) you can post to and check the mailing list without actually being a member. For access to the newsserver you need the following information:
Username: Faction number
Password: Faction password
